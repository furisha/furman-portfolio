About Me

My first IT carrer job was QA automation tester. Project that I mostly worked on was in-house project.

The basis of the application we developed was to get/convert a large amount of data and helps drive competitive advantage with predictive QA visual analytics and dashboards across popular application lifecycle management software testing tools such as: JIRA, Microsoft Excel, Tricentis Tosca , Spira Test, Sonarqube, Tasktop. In short, the application helps deliver software on time with less risk

The methodology we used during software development was Agile-Scrum. We were guided by sprints that lasted 1 or 2 weeks with all ceremonies included.

Application was based on PhP, React.js and Angular.js, and we perform end-to-end testing using Tosca testing tool, which is btw a great tool.

Working on this project required knowledge of: GIT version control service, CI tools like Jenkins, PM tool like JIRA and Tosca software testing tool as well.

The main types of testing we used during development were: Functional, UI, Regression and E2E.

During the work on the project, I became well acquainted with terms such as STLC, Agile/Scrum Methodology, and also what is a team spirit, team management, how to maintenance and create test suites and test cases according to the requiremants and specifications and much more.

I became very interested in test automation and the problem I saw after couple of years was working in a huge team and limitation of using only one testing tool. This limitation led me to research and learn how to use as many testing tools as possible and to recognize myself in a particular programming language.

One of the biggest adventages of test automation is that gives a huge confidence to the whole development team and shorten the time of QA's for regression or e2e testing of complex and large applications.

My learning stages were mostly in step with the needs of the clients, but I used to have complete freedom of choice which technology and tools to use for automation testig.

At the beginning, I decided to focus my learning on JS and OOP concepts. After I mastered a Cypress testing tool where I learned what Page Object Model is and how and when POM is good or bad to be used, how to intercept an API requests and assert results, and basically all the features which this tool adorns.

Afterwords comes Java, Pyhon, Selenium and BDD with Gherkin language as well.

Cypress was the starter because its open source, legerity of use, simple for understanding automation process and debugging, intercepting API requests, coding, reporting, and other cool features.

Until now I worked on several different projects. Projects and applications varied from outsourced to in-house products. So far I have worked on about 10 different projects that were quite different if take into account their management, methodologies and testing aproaches used during development phases.

The customers I saved from bugs and secured from security breaches in production were mostly MAC/WIN OS oriented.

So far I have worked on the development of various applications related to: medicine, management, recruitment, matching, entertainment, forms, etc.

I am currently a QA team leader of two QA warriors, with whom I am working on a project based on Django, Python and MySql. QA tests are written in Selenium using PyTest and implemented in the continuous integration process.

I found myself more efficency when work remotely within smaller and quieter environment where I can get better focus.
